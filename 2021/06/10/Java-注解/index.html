注解

一. 自定义注解

  • 格式

  • 本质：注解本质就是一个接口，该接口默认继承了Annotation接口

  • 属性：接口中的抽象方法

    • 要求

      1. 属性的返回值类型：

        • 基本数据类型
        • String
        • 枚举
        • 注解
        • 以上类型的数组

      2. 定义了属性，在使用时需要给属性赋值

        • 如果定义属性时，使用default关键字给属性默认初始化值，则使用注解时，可以不进行属性的赋值
        • 如果只有一个属性需要赋值，并且属性名称为value时，在赋值时可以省略属性名称
        • 数组赋值时，值使用{ }包裹，如果数组中只有一个值的话，则{ }可以省略不写

二. 元注解

用于描述注解的注解，JDK已经定义好，可以在我们自定义注解的时候使用

  • @Target：描述注解能够作用的位置

  • @Retention：描述注解被保留的阶段

    1. @Retention(RetentionPolicy.RUNTIME)
自定义注解通常使用这个，表示当前被描述的注解，会保留到class字节码文件中，并被JVM读取到

  • @Documented：描述注解是否被抽取到API文档中

    被此注解标记的自定义注解，在使用自定义注解标注其他代码时，生成的API文档会显示自定义注解，没有被@Document标记的自定义注解是不会生成自定义注解的

  • @Inherited：描述注解是否被子类继承，如果添加此注解，父类中的注解会自动被子类继承

三. 注解的使用（解析）

获取注解中定义的属性值

  1. 获取注解使用位置的对象，包括（Class, Method, Field），Class arc = AnoReflect.class
  2. 使用字节码对象arc的getAnnotation(Pro.class)的方法获取指定的实例化注解对象
  3. 调用注解中的抽象方法获取配置的属性值className，methodName
  4. 使用Class类的静态方法forName(String className)，将属性值对应的类加载进内存得到一个运行时对象aClass
  5. 调用运行时对象aClass的newInstance()方法在堆内存生成一个具体的对象Object o
  6. 调用运行时对象aClass的getMethod(methodName)获取到对象的方法对象Method method
  7. method.invoke(Object o)，方法对象的invoke方法执行指定对象的方法对象代表的方法
