一. 数据库的基本概念

  1. 英文名称：DataBase（DB）
  2. 用于存储和管理数据的仓库
  3. 数据库的特点
    1. 持久化存储数据，数据库就是一个文件系统
    2. 方便的存储和管理
    3. 使用统一的方式来操作数据库 — SQL
  4. 常见的数据库软件

二. MySql数据库软件

  1. mysql的启动

    1. 使用管理员权限打开cmd

    1. mysql -uroot -proot，连接本地的Mysql
    2. mysql -h127.0.0.1(ip) -uroot -proot，连接对应地址主机上的Mysql
    3. mysql –host = 127.0.0.1(ip) –user = root –password = root

  3. 退出

    1. exit
    2. quit

三. SQL

  1. 什么是SQL：Structured Query Language(结构化查询语言)，其实就是定义了操作所有关系型数据库的规则

  2. SQL通用的语法

    1. SQL语句可以单行或者多行书写，以封号结尾
    2. 可以使用空格和TAB缩进增强语句的可读性。
    3. MySQL的数据库SQL语句不区分大小写，关键字建议使用大写。
    4. 3种注释
      • 单行注释：– 注释内容 或者 # 注释内容（mysql特有）;（–后必须加空格，#后不用加空格）
      • 多行注释：/*注释*/

  3. SQL语句的分类

    1. DDL(Definition)：数据定义语言，用来定义数据库对象：数据库，表，列等。关键字：create，drop，alter等。
    2. DCL(Control)：数据控制语言，用来定义数据库的访问权限和安全级别，及创建用户。关键字：GRANT, REVOKE等。
    3. DML：增删改表中的数据。关键字：insert, delete, update等。
    4. DQL：查询表中的数据。关键字：select, where等。

四. DDL（Data Definition Language）

  1. 操作数据库：CRUD

    1. C(Create)：创建
      • create database 数据库名称
      • create database if not exists 数据库名称，//如果不存在该数据库再创建
      • create database 数据库名称 character set GBK，//以指定的字符集创建数据库
      • create database if not exists 数据库名称 character set GBK
    2. R(Retrieve)：查询
      • 查询所有数据库的名称：show databases;
      • 查询某个数据库的字符集和创建语句：show create database 数据库名称;
    3. U(Update)：修改
      • alter database 数据库名称 character set 字符集名称 //修改数据库的字符集
    4. D(Delete)：删除
      • drop database 数据库名称 //删除数据库
      • drop database if exists 数据库名称，//如果该数据库存在再删除
    5. 使用数据库
      • select database() //查询当前正在使用的数据库
      • use 数据库名称 //进入该数据库文件夹

  2. 操作表

    1. C(Create)：创建

      • 语法

      • mysql中常用的数据类型：

        1. int，整数类型 //eg. age int，
        2. double，小数类型 // score double(5, 2)，小数最多5位，保留后两位
        3. 日期类型
          • date，只包含年月日，格式为 yyyy-MM-dd
          • datetime, 包涵年月日时分秒， 格式为 yyyy-MM-dd HH:mm:ss
          • timestamp, 时间戳类型，包涵年月日时分秒， 格式为 yyyy-MM-dd HH:mm:ss，如果将来不给这个字段赋值，或赋值为NULL，则默认使用当前系统时间值
        1. varchar，字符串类型 //eg. name varchar(20) 最大长度20个字符

    2. R(Retrieve)：查询

      • show tables； //查询某个数据库中所有表的名称
      • desc 表名；//查询表结构

    3. U(Update)：修改

      • alter table 表名 rename to 新的表名 //修改表名
      • alter table 表名 character set utf8 //修改表的字符集 ；show create table 表名（查看表的字符集）
      • alter table 表名 add 列名 数据类型； //添加一列
      • alter table 表名 drop 列名 //删除列
      • alter table 表名 change 旧列名 新列名 新数据类型 //修改列名称，类型；
      • alter table 表名 modify 列名 新数据类型 //修改列名的数据类型；

    4. D(Delete)：删除

      • drop table 表名 //删除表
      • drop table if exists 表名 //如果存在该表，则删除
      • truncate table 表名 //删除表，然后再创建一个同样格式的空表

五. DML：增删改表中的数据

  1. 添加数据：insert into 表名(列名1， 列名2， …，列名n) values(值1，值2，…，值n)；
    • 注意事项：
      1. 列名和值要一一对应
      2. 如果表名后，不定义列名，则默认给所有列添加值
      3. 除了数字类型，其他类型都需要使用引号（单双都可以）包裹起来
      4. 如果某一项为主键primary key，对应的添加数据设置为null即可。
  1. 删除数据：delete from 表名 [where 条件]
    • 注意事项：不写条件，删除表中所有数据，有多少条数据，语句就会执行多少次
  2. 修改数据：update 表名 set 列名1 = 值1，列名2 = 值2，…，列名n = 值n [where 条件]
    • 注意事项：如果不写条件，会将表中所有的数据修改

六. DQL：查询

  1. 语法：select 字段列表 from 表名列表 where 条件列表 group by 分组字段 having 分组之后的条件 order by 排序 limit 分页限定
  2. 基础查询：
    1. 多个字段的查询
      1. 去除重复：select distinct 字段名1，字段名2，… from 表名；
      2. select 字段名1，字段名2，… from 表名； select * from 表名，查询所有字段
    2. 计算列：select 字段名1, 字段名2, 字段名3, ifnull(字段名2, 0) + IFNULL(字段名3, 0) from tes;
      1. 一般使用四则运算计算列的数值
      2. ifnull(字段名2, exp2) 如果改字段的某一行值为null时，将值替换为exp2的值，null参与的运算结果都为null
    3. 别名
      1. as
      2. as省略用空格替代
  3. 条件查询：
    1. where 子句后跟条件
    2. 运算符：
      • >、<、<=、>、=、<> <>在 SQL 中表示不等于，在 mysql 中也可以使用!=, 没有==
      • between…and , 在一个范围之内，如：between 100 and 200 相当于条件在 100 到 200 之间，包头又包尾( [100, 200] )
      • IN(集合), 集合表示多个值，使用逗号分隔
      • LIKE ‘’张%’；LIKE ‘’张_’ , 模糊查询 //‘%’ 表示多个任意字符 ，’ _‘表是单个任意字符
      • IS NULL；is not null, 查询某一列为 NULL 的值，注：不能写=NULL
      • and 或 && ，与，SQL 中建议使用前者，后者并不通用。
      • or 或 ||
      • not 或 !

  1. 排序查询：order by 排序字段1 排序方式1，排序字段2 排序方式2，…，排序字段n 排序方式n;
    1. ASC：升序，默认的
    2. DESC：降序
    3. 如果有多个排序条件，当前面的条件值一样时，才会判断后边的条件
  1. 聚合函数：将一列数据作为一个整体，进行纵向的计算，比如一个记录班级学生成绩的表，计算某一学科的班平均成绩
    1. count：计算个数
    2. max：计算最大值
    3. min：计算最小值
    4. sum：计算和
    5. avg：计算平均值
  1. 分组查询
    1. 语法：group by 分组字段;
    2. 注意：
      • 分组之后可查询的字段：分组字段，聚合函数
      • where 和 having 的区别：where 再分组之前进行限定，如果不满足条件，则不参与分组。having 在分组之后进行限定，如果不满足，则不会被查询出来
      • where 后不可跟聚合函数，having 可以进行聚合函数的判断

  1. 分页查询
    1. 语法：limit 开始的索引，每页查询的条数
    2. 开始的索引 = （当前页码数 - 1）* 每页显示的条数
    3. 每个数据库的分页查询语句不同

  1. 多表查询：

    1. 笛卡尔积

      • 有两个集合A，B，选取两个集合的所有组成情况
      • 要完成多表查询，需要消除无用的数据

    2. 分类：

      1. 内连接查询：

        1. 隐式：

          • ~~~mysql
            SELECT
                t1.属性1, 
    t1.属性2, 
    t2.属性1
            FROM
                table1 AS t1, 
    table2 AS t2
            WHERE
                t1.外键名 = t2.主键名,
    条件n;

        2. 注意事项：

          1. 从那些表中查询数据
          2. 条件是什么
          3. 查询哪些字段

      2. 外连接查询

        1. 左外连接
          • 语法：select 字段列表 from 表1 left [outer] join 表2 on 条件
          • 查询左表的所有数据以及两表的交集部分
        2. 右外连接
          • 语法：select 字段列表 from 表1 right [outer] join 表2 on 条件
          • 查询右表的所有数据以及两表的交集部分

      3. 子查询

        1. 概念：查询中嵌套查询，称嵌套查询为子查询

        2. ~~~mysql
          select * from t1 where age = (select MAX(age) from t1);

  2. 非空约束：not null

    1. 创建表的时候添加约束
    2. 删除某一列的非空约束，alter table 表名 modify 列名 数据类型
    3. 添加非空约束：alter table 表名 modify 列名 数据类型 not null
  1. 唯一约束：unique
    1. 可以为null，但是只能有一个null值
    2. 给已存在的表添加唯一约束：
      • 新增一列：ALTER TABLE stu ADD phone VARCHAR(20) UNIQUE;
      • 给已存在的列增加约束：ALTER TABLE stu MODIFY id INT UNIQUE;
      • 已存在的列项有重复数据无法添加唯一约束，需要删除其中一条重复项
    3. 删除唯一约束：ALTER TABLE stu DROP INDEX 已经添加唯一约束的列名;
  1. 外键约束：foreign key，让表与表产生关系，从而保证数据的正确性，外键可以为null
    1. 在创建表时添加外键约束：constraint 自定义外键名 foreign key (外键列名称) references 主表名称(主表列名称)
    2. 删除：alter table 定义了外键的表名 drop foreign key 自定义外键名
    3. 添加：alter table 表名 add constraint 自定义外键名 foreign key (外键列名称) references 主表名称(主表列名称)
    4. 级联：
      1. 级联更新，在创建或修改语句后添加 on update cascade
      2. 级联删除，on delete cascade

八. 数据库的设计

  1. 多表之间的关系
    1. 一对一，人和身份证
    2. 一对多(多对一)，部门和员工
    3. 多队多，学生和课程
  2. 实现关系
    1. 一对多(多对一)：在多的一方建立外键，指向一的一方的主键
    2. 多对多：实现需要借助第三章中间表，中间表至少包涵两个字段，这两个字段作为第三张表的外键，分别指向两张表的主键
    3. 一对一：可以在任意一方添加唯一外键指向另一方主键

  1. 数据设计范式

    1. 概念：设计数据库时需要遵循的规范，设计关系数据库时，遵从不同的规范要求，设计出合理的关系型数据库，这些不同的规范要求被称为不同的范式，各种范式呈递次规范，越高的范式数据库冗余越小。

      目前关系数据库有六种范式：第一范式（1NF）、第二范式（2NF）、第三范式（3NF）、巴斯-科德范式（BCNF）、第四范式(4NF）和第五范式（5NF，又称完美范式）。

    2. 分类：

      1. 第一范式（1NF）：数据库表的每一列都是不可分割的原子数据项，而不能是集合，数组，记录等非原子数据项
      2. 第二范式（2NF）：在1NF的基础上，非码属性必须完全依赖于码（在1NF基础上消除非主属性对主码的部分函数依赖）
        1. 概念：
          • 函数依赖：A –> B，如果通过A属性（属性组）的值，可以确定唯一B属性的值，则称B依赖于A，学号–>姓名，（学号，课程）–>分数
          • 完全函数依赖：A –> B，如果A是一个属性组，B属性值的确定需要依赖A属性组中的所有属性值，则B完全依赖于A
          • 部分函数依赖：A –> B，如果A是一个属性组，B属性值的确定只需要依赖A属性组中的某一些属性值，则B部分依赖于A
          • 传递函数依赖：A–>B, B–>C，如果通过A属性（属性组）的值，可以确定唯一B属性的值，再通过B属性（属性组）的值,确定唯一属性C，则称C传递依赖于A。
          • 码：如果在一张表中，一个属性或属性组，被其他所有属性完全依赖，则称这个属性（属性组）为该表的码
            • 主属性：码属性组中的所有属性
            • 非主属性：其他除码属性组的属性
      3. 第三范式（3NF）：在2NF基础上，任何非主属性不依赖于其它非主属性（在2NF基础上消除传递依赖）

十. 事务

  1. 事务的基本介绍
    1. 概念：如果一个包涵多个步骤的业务操作，被事务管理，那么这些操作要么同时成功，要么同时失败
    2. 操作：
      1. 开启事务：start transaction
      2. 回滚：rollback
      3. 提交：commit
    3. mysql数据库中事务默认自动提交（oracle数据库默认手动提交）
      1. 事务提交的两种方式：
        1. 自动提交：mysql中的一条DML语句自动提交一次事务
        2. 手动提交：需要先开启事务，再提交
      2. 修改事务的默认提交方式
        1. 查看事务的默认提交方式：select @@autocommit；– 1代表自动提交 0代表手动提交
        2. 修改默认提交方式：select @@autocommit = 0；
  2. 事务的四大特征
    1. 原子性：事务是不可分隔的最小单位，要么同时成功，要么同时失败
    2. 持久性：事务一旦提交或者回滚，数据库会持久化的保存数据
    3. 隔离性：多个事务之间相互独立
    4. 一致性：事务操作前后数据总量不变
  3. 事务的隔离级别
    1. 概念：多个事务之间隔离，相互独立，但是如果多个事务操作同一批数据，则会引发一些问题，设置不同的隔离级别就可以解决这些问题。
    2. 存在问题：
      1. 脏读：一个事务读取到另外一个事务没有提交的数据
      2. 不可重复读（虚读）：在同一个事务中，两次读取到的数据不一样
      3. 幻读：一个事务（DML）操作数据表中所有的记录，另一个事务添加了一条数据，则第一个事务查询不到自己的修改
    3. 隔离级别，按顺序安全性越来越高，但是效率越来越低
      1. read uncommitted：读未提交 // 脏读，虚读，幻读问题都会产生
      2. read committed（oracle默认级别）：读已提交 //虚读， 幻读问题产生
      3. repeatable read（默认级别）：可重复读 //幻读问题产生
      4. serializable：串行化，一个事务操作表时，另外一个事务无法操作该表 //不会产生问题
    4. 查询隔离级别：select @@tx_isolation；
    5. 设置隔离级别：set global transaction isolation level 级别字符串

十一. DCL（管理用户，授权）

  1. 管理用户
    1. 添加用户：create user ‘用户名’@‘主机名’ identified by ‘密码’，主机名用通配符%替代，表示任意主机可访问
    2. 删除用户：drop user ‘用户名’@‘主机名’
    3. 修改用户密码 ：set password for ‘用户名’@‘主机名’ = password(‘新密码’) ；update语句也可以修改
      • 忘记root用户密码
        1. cmd —> net stop MySQL80
        2. mysqld –skip–grant–tables
        3. 开启另一个cmd，输入mysql回车可以直接登录
        4. use mysql; update user set password = password(‘你的新密码’) where user = ‘root ‘ 修改root用户密码，关闭所有cmd窗口
        5. 任务管理器结束mysqld进程
        6. cmd —> net star MySQL80
        7. 使用新密码登录就可以使用
    4. 查询用户：在mysql数据库中user表中存放用户数据
  2. 权限管理
    1. 查询权限：show grants for ‘用户名’@‘主机名’
    2. 授予权限：grant 权限列表 on 数据库名.表名 to ‘用户名’@‘主机名’
      • 授予用户所有数据库的所有表的所有权限：grant all on *.* to ‘用户名’@‘主机名’
    3. 撤销权限：revoke 权限列表 on 数据库名.表名 from ‘用户名’@‘主机名’
