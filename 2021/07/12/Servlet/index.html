一. 网络通信三要素

三. Servlet

  1. 概念：server applet 运行在服务器端的小程序；Servlet就是一个接口，定义了Java类被浏览器访问到(tomcat识别)的规则，我们需要自定义类实现Servlet接口
  2. 快速入门
    1. 创建JavaEE项目
    2. 定义一个类，实现Servlet接口：public class ServletDemo1 implements Servlet{}
    3. 实现接口中的配置方法
    4. 配置Servlet：在web.xml中配置
      • Servlet3.0之后的版本支持注解配置：@WebServlet("资源路径")：@WebServlet("/demo2")
  1. Servlet的执行原理
    1. 当服务器接收到客户端浏览器的请求之后，会解析请求的URL路径，获取访问的Servlet的资源路径
    2. 查找web.xml文件，是否有对应的<url-pattern>标签体内容符合虚拟路径
    3. 如果有，则找到对应的<servlet-clasee>全类名
    4. tomcat会将字节码文件加载进内存，并且创建其对象
    5. 调用类的方法
  2. Servlet中的生命周期
    1. public void init(ServletConfig servletConfig) throws ServletException {}
      1. 初始化方法，在Servlet被创建时执行，只会被执行一次，说明Servlet在内存中只存在一个对象，Servlet是单例的
      2. Servlet是单例的所存在的问题
        • 多个用户同时访问时，可能存在线程安全问题
        • 解决：尽量不要在Servlet中定义成员变量，即使定义了成员变量，也不要修改值
      3. 默认情况下，Servlet第一次被浏览器访问时，Servlet被创建
      4. 在web.xml文件内的<servlet>标签下配置
        • <load-on-startup>的内容为负整数：第一次被浏览器访问时创建
        • <load-on-startup>的内容为非负整数：服务器启动时创建
    2. public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {}
      • 提供服务的方法，每一次Servlet被访问时，都会执行此方法
    3. public void destroy() {}
      • Servlet服务器正常关闭时，执行此方法，执行一次，destroy()执行完毕，Servlet被销毁，一般用于释放资源
  3. IDEA和tomcat的相关配置
    1. IDEA会为每一个tomcat部署的项目单独建立一份配置文件
      • 查看控制台的log：Using CATALINA_BASE: “C:\Users\Avicii\AppData\Local\JetBrains\IntelliJIdea2021.1\tomcat\9cedbb32-7110-41e9-be81-9250d9a70149”

    3. WEB-INF文件夹内的html文档时不会被访问的，应该存放在webapp目录下

四. HTTP

  1. 概念：Hyper Text Transfer Protocol 超文本传输协议

    1. 传输协议：定义了客户端和服务器通信时，发送数据的格式
    2. 特点：
      • 基于TCP/IP的高级协议
      • 默认端口号：80
      • 基于请求/响应模型：一次请求对应一次响应
      • 无状态的协议：每次请求之间相互独立，不能交互数据
    3. 历史版本
      • 1.0：每一次请求响应都会建立新的连接
      • 1.1：复用连接

  2. 请求消息数据格式

    1. 请求行：请求方式 请求url 请求协议/版本（GET /login.html HTTP/1.1）

      • 请求方式：HTTP协议有7中常用的请求方式，常用的有两种
        • GET：请求参数在请求行中，在url后，请求的url长度是有限制的
        • POST：请求参数在请求体中，请求的url长度是没有限制的

    2. 请求头：客户端浏览器告诉服务器的一些信息

      • 格式：

        1. 请求头名称：请求头值

      • 常见的请求头

        1. User-Agent：访问服务器的当前浏览器的版本信息，服务器通过来获取此信息，来解决浏览器的兼容问题
        2. Referer: http://localhost:8080/login.html：告诉服务器，我(当前请求)从哪里来？
          • 作用
            1. 防盗链
            2. 统计工作

    3. 请求空行：分隔空行，用于分隔POST请求的请求头，和请求体

    4. 请求体(正文)：封装POST请求消息的请求参数

      • 格式
        1. username=zhangsan，get请求方式没有请求体，post有

    5. 字符串格式

      GET /login.html HTTP/1.1

      Host: localhost:8080

      Connection: keep-alive

      Upgrade-Insecure-Requests: 1

      User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36 Edg/91.0.864.67

      Accept:text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,/;q=0.8,application/signed-exchange;v=b3;q=0.9

      Accept-Encoding: gzip, deflate, br

      Accept-Language: zh-CN,zh;q=0.9,en;q=0.8,en-GB;q=0.7,en-US;q=0.6

      Referer: http://localhost:8080/login.html

      username=zhangsan

  3. 响应数据数据格式

    1. 数据格式：

      1. 响应行
        1. 组成：协议/版本 响应状态码 状态码描述
        2. 响应状态码：服务器告诉客户端浏览器本次请求和响应的一个状态
          1. 状态码都是三位数字
          2. 分类
            1. 1XX：服务器接受客户端消息，但没有接收完成，等待一段时间后，发送1XX
            2. 2XX：成功，代表：200
            3. 3XX：重定向，代表：302(重定向)，304(访问缓存)
            4. 4XX：客户端错误，代表：404(请求路径没有对应的资源)，405(请求方式没有对应的doXxx方法)
            5. 5XX：服务器端错误。代表：500(服务器内部出现异常，比如代码错误)
      2. 响应头
        1. 格式：头名称：值
        2. 常见响应头：
          1. Content-Type：服务器告诉客户端本次响应体数据格式以及编码格式
          2. Content-disposition：服务器告诉客户端以什么方式打开响应体数据
            1. 值：
              • in-line：默认值，在当前页面打开
              • attachment;filename=xxx：以附件的形式打开响应体。文件下载
      3. 响应空行
      4. 响应体

    2. 响应字符串格式

      HTTP/2.0 200 OK

      Conten-Type: text/html;charset=UTF-8

      Conten-Length:101

      Date:Wed,06 Jun 2018 07:09:33 GMT

五. Request

  1. request对象和response对象的原理

    1. request对象和response对象是由服务器创建的，我们来使用他们
    2. request对象是来获取请求消息，response对象是来设置响应消息的

  2. request对象继承体系结构

    1. ServletRequest：接口
    2. HttpServletRequest：继承ServletRequest的接口
    3. org.apache.catalina.connector.RequestFacade：实现HttpServletRequest的类

  3. request对象的功能：

    1. 获取请求消息

      1. 获取请求行数据

        1. 请求行数据：GET /day14/demo1?name=zhangsan HTTP/1.1
        2. 方法：
          • 获取请求方式(GET)：String getMethod();
          • 获取虚拟路径(/day14)：String getContextPath();
          • 获取Servlet路径(/demo1)：String getServletPath();
          • 获取请求的URI(/day14/demo1)：String getRequestURI();统一资源标识符
          • 获取请求的URL(http://localhost/day14/demo1)：StringBuffer getRequestURL();统一资源定位符
          • 获取GET方式请求参数(name=zhangsan)：String getQueryString();
          • 获取协议及版本(HTTP/1.1)：String getProtocol();
          • 获取客户机的ip地址：String getRemoteAddr();

      2. 获取请求头数据

        1. 方法：
          • String getHeader(String name)：通过请求头的名称获取请求头的值，请求头的名称不区分大小写
          • Enumeration getHeaderNames()：获取所有请求头名称
            1. boolean new Enumeration().hasMoreElements()：
            2. String name new Enumeration().nextElements()：获取请求头的名称

      3. 获取请求体数据

        1. 请求体：只有POST请求方式，才有请求体，请求体中封装了POST请求的请求参数

        2. 步骤：

          1. 获取流对象

            • BufferedReader request.getReader()：获取字符输入流，只能操作字符数据

            • ServletInputStreamrequest.getInputStream()：获取字节输入流，可以操作所有类型数据

          2. 再从流对象中拿数据

    2. 其他功能：

      1. 获取请求参数的通用方式，不论通过get还是post请求方式都可以使用下列方法来获取请求参数

        • String getParameter(String name)：根据参数名称获取参数值 username=zhangsan&password=123

        • String[] getParameterValues(String name)：根据参数名称获取参数值的数组

          hobby=music&hobby=game

        • Enumeration getParameterNames()：获取所有请求的参数名称

        • Map<String String[]> getParameterMap()：获取所有参数的map集合

        • 中文乱码问题

          • get方式：tomcat8已经将get方式的乱码问题解决了
          • post方式：会乱码，在获取参数前设置request的编码 request.setCharacterEncoding(“网页编码格式”);

      2. 请求转发：一种在服务器内部的资源跳转方式

        1. 步骤：

          1. 通过request对象获取请求转发器对象：

            RequestDispatcher getRequestDispatcher(String path)

          2. 使用RequestDispatcher对象进行转发：

            forward(ServletRequest request, ServletResponse response);

            eg: req.getRequestDispatcher("/register").forward(req, resp);

        2. 特点：forward

          1. ==浏览器地址栏路径不发生变化==
          2. ==只能转发到当前服务器内部资源中==
          3. ==转发是请求一次==

      3. 共享数据

        1. 域对象：一个有作用范围的对象，可以在范围内共享数据
        2. request域：代表一次请求的范围，一般用于请求转发的多个资源中共享数据
        3. 方法:
          1. void setAttribute(String name, Object obj)：存储数据
          2. Object getAttribute(String name)：通过键获取数据
          3. void removeAttribute(String name)：通过键删除数据

      4. 获取ServletContext

        1. 方法：ServletContext getServletContext()

六. Response

  1. 功能：设置响应消息

    1. 设置响应行
      1. 格式：HTTP/1.1 200 ok
      2. 设置状态码：setStatus(int sc)
    2. 设置响应头：setHeader(String name, String Value)
    3. 设置响应体：
      1. 使用步骤
        1. 获取输出流：
          • 字符输出流：PrintWriter getWriter()
          • 字节输出流：ServletOutputStream getOutputStream()
        2. 使用输出流，将数据输出到客户端浏览器

  2. 完成重定向，服务器内部资源跳转方式

    1. 特点：redirect

      1. ==地址栏发生变化==
      2. ==重定向可以访问其他站点(服务器)的资源==
      3. ==重定向是两次请求，不能用request域对象来共享数据==

    2. 路径的写法

      1. 路径的分类
        1. 相对路径：提供相对路径不可以确定唯一资源
          1. 如：./index.html
          2. 不以/开头，以.开头的路径
          3. 规则：找到当前资源和目标资源之间的相对位置关系，./当前目录，可以省略；../后退一级目录
        2. 绝对路径：提供绝对路径可以确定唯一资源
          1. 如：http://localhost:8080/cc/responseDemo2 简写为 /cc/responseDemo2
          2. 以/开头的路径
          3. 规则：判断定义的路径是给谁用的？判断请求将从哪发出
            1. 给客户端浏览器使用：需要加虚拟路径(项目的访问路径)
              1. 建议虚拟路径动态获取：request.getContextPath();
              2. <a>，<form>，重定向…
            2. 给服务器用，不需要加虚拟路径
              1. 转发路径

    3. 服务器输出字符数据到浏览器

      1. 步骤
        1. 获取字符输出流
        2. 输出数据
      2. 乱码问题
        1. PrintWriter pw = response.getWriter(); 获取的流的默认编码是ISO-8859-1
        2. 设置改流的默认编码
        3. 告诉浏览器响应体使用的编码：简单的设置编码的方式
          response.setContentType(“text/html;charset=utf-8”);

    4. 服务器输出字节数据到浏览器

    5. 验证码

      1. 本质：图片
      2. 目的：防止恶意的表单注册
      3. 点击图片和超链接切换验证码

七. ServletContext对象

  1. 概念：代表整个web应用，可以和程序的容器(服务器)来通信

  2. 获取

    1. 提供request对象来获取：request.getServletContext();
    2. 提供HttpServlet获取：this.getServletContext();

  3. 功能：

    1. 获取MIME类型

      1. MiME类型：在互联网通信过程中定义的一种文件数据类型； 格式：大类型/小类型 eg：text/html，image/jepg
      2. 获取方法：String getMimeType(String file)

    2. 域对象：共享数据

      1. void setAttribute(String name, Object obj)：存储数据
      2. Object getAttribute(String name)：通过键获取数据
      3. void removeAttribute(String name)：通过键删除数据
      4. ServletContext对象的范围：共享所有用户所有请求的数据

    3. 获取文件的真是(服务器)路径：String getRealPath(String pasth);

      1. String aPath = servletContext.getRealPath(“/a.text”) //web目录下的资源访问
      2. String bPath = servletContext.getRealPath(“/WEB-INF/b.text”) //WEB-INF目录下的资源访问
      3. String cPath = servletContext.getRealPath(“/WEB-INF/classes/c.text”) //java目录下的资源访问
