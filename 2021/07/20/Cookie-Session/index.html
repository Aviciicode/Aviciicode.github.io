二. 会话技术

  1. 会话：一次会话中包含着多次请求和响应
    • 一次会话中：浏览器第一次给服务器发送资源请求时，会话建立，直到由一方断开为止
  2. 功能：在一次会话范围内的多次请求间，共享数据
  3. 方式：
    1. 客户端会话技术：Cookie
    2. 服务器端会话技术：Session

三. 客户端会话技术

  1. 概念：客户端会话技术，将数据保存到客户端
  2. 快速入门：
    1. 使用步骤：
      1. 创建Cookie类对象，绑定数据：new Cookie(String name, String value)
      2. 发送Cookie：response.addCookie(Cookie ccokie)
      3. 获取Cookie，拿到数据：Cookie[] request.getCookies();
  3. 实现原理：基于响应头set-cookie和请求头cookie实现

  1. cookie细节

    1. 一次可不可以发送多个cookie？

      • 可以，可以创建多个Cookie对象，然后使用 response.addCookie(Cookie ccokie)方法依次发送即可

    2. cookie在浏览器中保存多长时间？

      1. 默认情况下，当浏览器关闭后，cookie数据被销毁
      2. 持久化存储：
        1. new Coookie(key, value).setMaxAge(int seconds)方法的参数状态：
          1. 正数：将cookie数据写到硬盘的文件中。持久化存储，参数标识存活的秒数。
          2. 负数：默认值浏览器关闭销毁。
          3. 零：服务器删除cookie信息。

    3. cookie能不能存中文信息？

      1. 在tomcat 8 之前，cookie中不能直接存储中文数据，需要将中文数据转码(一般采用URL编码（%E3）)。
        1. 编码方法：String str = URLEncoder.encode(string, “utf-8”)
        2. 解码方法：String str = URLDecoder.decode(string, “utf-8”)
      2. 在tomcat 8 之后，cookie支持中文数据。

    4. cookie获取的范围是多大？

      1. 假设在一个tomcat服务器中，部署了多个web项目，那么在这些web项目中cookie能不能共享？
        • 默认情况下cookie不能共享
        • new Coookie(key, value).setPath(String path)：设置cookie的获取范围。默认情况下，设置为当前项目的虚拟目录；如果要共享，可将path设置为 “/“
      2. 不同的tomcat服务器间cookie共享问题？
        • new Coookie(key, value).setDomain(String path)：如果设置的一级域名相同，那么多个服务器之间cookie可以共享
          • setDomain(“.baidu.com”)，那么tieba.baidu.com和news.baidu.com中cookie可以共享

  2. Cookie的特点和作用

    1. cookie存储数据在客户端浏览器
    2. 浏览器对于单个cookie的大小有限制(4kb)，以及同一个域名下的总cookie数量也有限制(20)
    3. 作用：
      1. cookie一般用于存储少量的不太敏感的数据
      2. 在不登陆的情况下，完成服务器对客户端的身份识别

  3. 案例：记住上一次访问时间

  4. 需求：

    1. 访问一个Servlet，如果是第一次访问，则提示：您好，欢迎您首次访问。
    2. 如果不是第一次访问，则提示：欢迎回来，您上次访问时间为:显示时间字符串

  5. 分析：

    1. 可以采用Cookie来完成
    2. 在服务器中的Servlet判断是否有一个名为lastTime的cookie
         1. 有：不是第一次访问
       1. 响应数据：欢迎回来，您上次访问时间为:2018年6月10日11:50:20
       2. 写回Cookie：lastTime=2018年6月10日11:50:01
   2. 没有：是第一次访问
       1. 响应数据：您好，欢迎您首次访问
       2. 写回Cookie：lastTime=2018年6月10日11:50:01

  6. 注意：

    1. tomcat 8 以后cookie可以支持中文信息，但是特殊字符仍不支持，仍需要进行编码格和解码。下面图片中提示cookie中存在无效的字符值32，对应的就是ASCII码表32的空格字符。

四. JSP

  1. 概念：Java Server Pages，java服务器端页面。可以理解为一个特殊的页面，其中既可以指定定义html标签，又可以指定java代码，目的是为了简化书写

  2. 原理：

    1. JSP本质上就是一个Servlet
    2. 浏览器访问服务器端JSP文件的步骤
      1. 浏览器请求localhost/tomcat/index.jsp
      2. 服务器解析请求消息，找到是否有index.jsp资源。
      3. 如果找到了，会将index.jsp文件转换为 .java文件（index_jsp.java)
      4. 编译 .java文件，生成 .classs文件（index_jsp.class）
      5. index_jsp.class 文件就是一个Servlet，与浏览器进行交互

  3. JSP脚本：

    1. 概念：JSP定义java代码的方式
    2. <% java代码 %>：定义的java代码，在service方法中。service方法中可以定义什么，改脚本中就可以定义什么。
    3. <%！ java代码 %>：定义的java代码，在jsp转换后的java类的成员位置，比如成员变量或成员方法。
    4. <%= java代码 %>：定义在service方法中的代码，会输出到页面上。输出语句可以定义什么，该脚本中就可以定义什么。在jsp中定义<%= hello %>，会在class文件中的service方法中生成out.print("hello");语句。

  4. 注释：

    1. html注释
      • <!-- -->：只能注释html代码片段
    2. jsp注释（推荐使用）
      • <%-- --%>：可以注释所有

  5. JSP的内置对象：

    1. 概念：在jsp页面中不需要获取和创建，可以直接使用的对象

    2. jsp一般9个内置对象：

      1. request
      2. response
      3. out：字符输出流对象，可以将数据输出到浏览器页面上；和response.getWriter()类似
        1. response.getWriter.write()和out.write()的区别：
          1. 在tomcat服务器真正给客户端做出响应之前，会先找response缓冲区数据，再找out缓冲区数据。
          2. response.getWriter.write()数据输出永远在out.write()之前
          3. 一般统一使用out.write()，乱用会打乱页面布局
    3. 变量名 真实类型 作用
      pageContext PageContext 当前页面共享数据，还可以获取其他八个内置对象
      request HttpServletRequest 一次请求访问的多个资源(转发)
      session HttpSession 一次会话的多个请求间
      application ServletContext 所有用户共享数据
      response HttpServletResponse 响应对象
      page Object 当前页面(Serclet)的对象 this
      out JspWriter 输出对象，数据输出到页面上
      config ServletConfig Servlet的配置对象
      exception Throwable 异常对象

  6. JSP的指令：

    1. 作用：用于配置JSP页面，导入资源文件

    2. 格式：<%@ 指令名称 属性名1=属性值1 属性名2=属性值2 ... %>

    3. 分类：

      1. page：配置JSP页面的

        1. contentType：等同于response.setContentType
          1. 设置响应体的mime类型以及字符集
          2. 设置当前JSP页面的编码方式（只能高级的开发工具才能生效，如果使用低级工具，则需要设置pageEncoding属性设置当前页面的字符集）
        2. import：导包
        3. errorPage：当前页面发生异常后，会自动跳转到指定的错误页面
        4. isErrorPage：标识当前页面是否是错误页面
          1. true：是，可以使用内置对象Exception
          2. false：否，默认值，不可以使用内置对象Exception

      2. include：包含页面的，导入页面的资源文件

        1. <%@incluede file=”top.jsp” %>

      3. taglib：导入资源

五. Session

  1. 概念：服务器端会话技术，在一次会话的多次请求间共享数据，将数据保存在服务器端的对象HttpSession中

  2. 快速入门：

    1. 获取HttpSession对象：HttpSession session = request.getSession();
    2. 使用HttpSession对象：
      1. void setAttribute(String name, Object value)
      2. Object getAttribute(String name)
      3. void removeAttribute(String name)

  3. 原理：Session的实现是依赖于Cookie的

    1. 浏览器第一次访问服务器，服务器创建一个session对象，并给session对象分配一个id，假如id=34224231
    2. 服务器会把响应头 set-cookie：JSESSIONID=34224231 响应给浏览器
    3. 浏览器在获取cookie之后，在之后的访问中都会把 cookie：JSESSIONID=34224231 发送给服务器
    4. 服务器之后再次获取session对象时，会根据这个cookie传递的id值，进服务器内存中查找这个session对象

  4. 细节：

    1. 当客户端关闭后，服务器不关闭，两次获取的session是否为同一个？

      1. 默认情况下不是的

      2. 如果需要相同，则可以创建Cookie，键为JSESSIONID，设置最大存活时间，让cookie持久化保存

    2. 客户端不关闭，服务器关闭后，两次获取的session是否为同一个？

      1. 服务器关闭，内存释放，不是同一个，但是要确保数据不丢失，tomcat自动实现钝化和活化的步骤
        1. session的钝化：在服务器正常关闭之前，将session对象序列化到硬盘上
        2. session的活化：在服务器启动后，将session文件转化为内存中的session对象即可

    3. session的什么时候被销毁？

      1. 服务器关闭

      2. session对象调用invalidate()方法

      3. session默认失效时间，30min，可以手动在web.xml文件中修改参数

  5. 特点：

    1. session用于存储一次会话的多次请求的数据，存储在服务器。
    2. session可以存储任意类型和任意大小的数据
    3. session于cookie的区别：
      1. session存储数据在服务器端，cookie在客户端
      2. session没有数据大小限制，cookie有
      3. session数据安全，cookie相对于不安全

  6. 案列：

    1. 案例需求：
      1. 访问带有验证码的登录页面login.jsp
      2. 用户输入用户名，密码以及验证码。

六. MVC开发模式

  1. JSP演变历史：
    1. 早期只有Servlet，只能使用response输出标签数据，非常麻烦
    2. 后来有jsp，简化了Servlet的开发，如果过度使用jsp，在jsp中书写大量的java代码，又写html的，造成难于维护，难于分工协作
    3. 再后来，java的web发展，借鉴mvc的开发模式，是的程序的设计更加合理性
  2. MVC：
    1. M：Model，模型。JavaBean
      1. 完成具体的业务操作，如：查询数据库，封装对象
    2. V：View，视图。JSP
      1. 展示数据
    3. C：Controller，控制器。Servlet
      1. 获取用户的输入
      2. 调用模型
      3. 将数据交给视图进行展示
  3. 优缺点：
    1. 优点：
      1. 耦合性低，方便维护，有利于分工协作
      2. 重用性高
    2. 缺点：
      1. 使得项目架构变得复杂，对开发人员要求高

七. EL表达式

  1. 概念：Expression Language 表达式语言

  2. 作用：替换和简化jsp页面中java代码的编写

  3. 语法：${表达式}

  4. 注意：JSP默认支持EL表达式，如果要忽略EL表达式

    1. 设置jsp中page指令中：isELIgnored=”true”，忽略当前jsp页面中所有的el表达式
    2. \${表达式}：忽略当前这个EL表达式

  5. 作用：

    1. 运算

      • 运算符：
        1. 算数运算符：+ - * /(div) %(mod)
        2. 比较运算符：> < >= <= == !=
        3. 逻辑运算符：&&(and) ||(or) !(not)
        4. 空运算符：empty，用于判断字符串，集合，数组对象是否为null或者长度是否为0；not empty则相反。${empty list},判断集合是否为空

    2. 获取值

      1. el表达只能从域对象中获取值

      2. 语法：

        1. ${域名称.键名}：从指定域中获取指定键的值

          1. 域名称：
            1. pageScope –> pageContext
            2. requestScope –> request
            3. sessionScope –> session
            4. applicationScope –> application(ServletContext)
          2. eg. 在request域中存储了name=张三，获取：${requestScope.name}

        2. ${键名}：表示依次从最小的域中查找是否有该键对应的值，知道找到为止

        3. 获取对象，List，Map集合的值

          1. 方法：对象，${域名称.键名.属性名} / ${键名.属性名}，本质是调用对象的getter方法。(逻辑视图)

          2. List集合：${域名称.键名[索引]}

          3. Map集合：${域名称.键名.key名称} / ${域名称.键名["key名称"]}

  6. 隐式对象：

    1. el中有11个隐式对象
    2. pageContext：获取其他8个内置对象，${pageContext.request.contextPath} 动态获取虚拟目录

八. JSTL标签

  1. 概念：JavaServlet Pages Tag Library，JSP标准标签库，是由Apache组织提供的开源免费的jsp标签。

  2. 作用：用于简化和替换jsp页面上的java代码

  3. 使用步骤：

    1. 导入jstl相关jar包
    2. 引入标签库：taglib指令：<%@ taglib %>
    3. 使用标签

  4. 常用的JSTL标签：

    1. if（相当于java代码的if语句）：<c:if test="boolean"></c:if> / <c:if test="${not empty list}"></c:if>

      1. test为必须属性，接收一个布尔类型的数据，如果为true则显示标签体内容，为false不显示标签体内容（不发送代码给浏览器渲染）
      2. 一般情况下，test属性值会结合el表达式一起使用
      3. if标签没有else情况，想要else情况，则可以在定义一个标签

    2. choose（相当于java代码的switch语句）：

    3. foreach（相当于java代码的java语句）：

      1. 完成重复的操作

        1. 属性：
          1. begin：开始值
          2. end：结束值(包含)
          3. var：临时变量
          4. step：步长
          5. varStatus：循环状态对象
            • index属性：容器中元素的索引，从0开始
            • count属性：循环次数，从1开始

      2. 遍历容器

        1. 属性：
          1. items：容器对象
          2. var：容器中元素的临时变量
          3. varStatus：循环状态对象
            • index属性：容器中元素的索引，从0开始
            • count属性：循环次数，从1开始
